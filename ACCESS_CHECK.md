# Access Check & Safety

Ναι, υπάρχει πάντα πιθανότητα ανθρώπινου λάθους όταν γίνονται αλλαγές σε repository.

Για να μειωθεί ο κίνδυνος:
- Κάθε αλλαγή γίνεται σε commit (ιστορικό και δυνατότητα επαναφοράς).
- Πριν το commit γίνεται έλεγχος με `git status` και review των διαφορών.
- Αν κάτι πάει στραβά, μπορούμε να κάνουμε revert σε προηγούμενο commit.
- Οι αλλαγές περνάνε από pull request ώστε να υπάρχει επιπλέον έλεγχος.

Συμπέρασμα: το ρίσκο δεν είναι μηδενικό, αλλά είναι ελεγχόμενο και αναστρέψιμο με σωστή ροή git.
